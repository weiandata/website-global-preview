# 37 · What Should Data Look Like to Feed IRT?

> Many people get stuck at the first step: with a pile of survey data, how should they organize it to start analysis? This article explains what kind of data IRT expects, as well as the most common data preparation pitfalls.

## The Data IRT Wants: A "Person × Item" Table

The most basic data format for IRT is a clean table (imagine an Excel sheet):

- **Each row represents a person** (a respondent, a survey form).
- **Each column represents an item**.
- **Each cell contains the score for that person on that item**.

For example, for a true/false question, the cell contains 1 (correct) or 0 (incorrect):

| Respondent | Item 1 | Item 2 | Item 3 | Item 4 | …   |
| ---------- | ------ | ------ | ------ | ------ | --- |
| Zhang San  | 1      | 0      | 1      | 1      | …   |
| Li Si      | 0      | 0      | 1      | 0      | …   |
| Wang Wu    | 1      | 1      | 1      | 0      | …   |

That's how simple it is. **Rows are people, columns are items, and cells are scores** — remember this "person × item matrix," and you've grasped the skeleton of IRT data.

## What to Put in Each Cell Depending on the Item Type

- **True/False Items (Binary Scoring)**: Put 0 or 1. 1 = correct, 0 = incorrect.
- **Likert Scale Items (Ordered Polytomous)**: Put the corresponding numerical value. For example, a five-point scale "Very Disagree → Very Agree" can be coded as 1, 2, 3, 4, 5 (or 0–4).
- **Partially Scored Items**: Put the actual score, for example, if an item is worth 3 points, put 0, 1, 2, 3.

**Key**: The numbers in the cells must be **meaningful scores or category codes**, and the coding rules for the same column (same item) must be consistent.

## The Three Most Common Data Preparation Pitfalls

**Pitfall 1: Reverse-Scoring Items Not Handled.**
In scales, there are often "reverse-scored items." For example, in a questionnaire measuring "happiness," most items are "I feel happy" (higher agreement means higher score), but there might be one item like "I often feel depressed" (higher agreement actually means lower happiness).

If you code this reverse-scored item as is, it will be "in the opposite direction" compared to other items, severely disrupting the analysis (this is a common cause of negative discrimination, which will be discussed in Chapter 46). **You must first reverse-score these items** (e.g., swap 1 ↔ 5, 2 ↔ 4 in a five-point scale), so that all items are in the same direction, before feeding the data to IRT. This is the easiest step to forget and has serious consequences. (Chapter Q29 in the existing content library specifically discusses reverse-scored items.)

**Pitfall 2: Filling Missing Values Arbitrarily.**
If a respondent didn't answer a question, the cell is empty (missing). **Do not arbitrarily fill it with 0!** In true/false items, "not answered" and "answered incorrectly" are two different things — treating "not answered" as "incorrect" is equivalent to arbitrarily deducting points, which distorts ability estimates. Missing values need to be handled properly and correctly (Chapter 38 will discuss this in detail), marked as "missing" rather than "0."

**Pitfall 3: Inconsistent Coding or Mixed Data.**
For the same item, some cells are filled with "1," others with "Correct," and others with "√"; or the table includes irrelevant columns such as names, notes, and dates. IRT only accepts clean, consistent, and purely numerical score matrices. **Before modeling, you must standardize the coding and remove irrelevant columns.**

## Where to Store Auxiliary Information

In addition to the score matrix, you often have other information: gender, urban/rural status, age, time taken to answer, etc. These should not be included in the score matrix but should be **stored separately and matched properly**, because they will be used later:

- Gender, urban/rural status, etc. → Used for DIF analysis (Chapter 23).
- Sampling weights → Used for estimating population distribution in large-scale surveys.
- Time taken to answer, etc. → May be referenced for identifying careless responding (Chapter 38).

**Scores go in the score matrix, background information goes in its own place, store them separately but ensure they can be linked** — this is a good habit for organizing data.

## An Often Overlooked Preparatory Step: A Data Dictionary

When data becomes large, you may easily forget what "Item 3" is, what 5 means, or what this variable represents. Therefore, the professional approach is to prepare a **data dictionary**: clearly document what each column represents, how it is coded, which items are reverse-scored, and how missing values are marked.

This document may seem trivial, but it can be life-saving months later (or when someone else takes over). **Good data is not only clean, but also understandable — it should explain why it is arranged that way.**

## One-Paragraph Summary

> **IRT wants a clean "person × item matrix": rows are people, columns are items, and cells are scores (put 0/1 for true/false, and the scale value for Likert items). Avoid the three main pitfalls: reverse-scored items must be reversed first, do not arbitrarily fill missing values with 0, and ensure consistent coding. Store background information separately for future use, and prepare a data dictionary. Only with well-organized data does meaningful analysis become possible.**
