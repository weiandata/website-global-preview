# A11 · Latent Regression: How Background Variables Enter the IRT Model Correctly

> IRT Advanced Series. A key component of large-scale assessment methodology, also a prerequisite for PV.

## Question: Estimating Scores First and Then Doing Regression, What's Wrong With This Approach?

Almost all assessment projects eventually need to answer this question: How is ability related to background variables such as family background, region, and teaching variables? The intuitive approach is a two-step process: first use IRT to estimate each person's ability θ̂, and then use θ̂ as the dependent variable in regression.

This "two-step approach" has a systematic flaw: **θ̂ is a point estimate with measurement error, and the nature of the error differs across estimation methods. Directly using it in regression leads to predictable directional bias.**

- Using **EAP/MLE point estimates** as the dependent variable: measurement error causes **regression coefficients to attenuate** (underestimate the effect of background variables), and the shrinkage property of EAP further distorts variance and subgroup differences — the fewer the items, the stronger the shrinkage, and the more the subgroup differences are underestimated;
- More subtly: EAP shrinkage is **shrinkage toward the prior mean**. If background information is not used in estimating ability, everyone shrinks toward the overall mean, and the true disadvantage of disadvantaged groups is "shrunk" — your fairness conclusions are directly contaminated by the method.

In matrix-based large-scale assessments (where each person only answers part of the items), individual measurement error is substantial, and the bias from the two-step approach cannot be ignored. This is the reason for the existence of latent regression.

## Solution: Bring Regression Into the Model

The idea behind latent regression is: **do not first estimate individual scores, but instead let the ability distribution itself be conditioned on background variables.** Model specification:

$$\theta_n \sim N(\beta_0 + \beta_1 x_{n1} + \dots + \beta_p x_{np},\ \sigma^2)$$

Response data and background variables **simultaneously** enter the likelihood, allowing for a single-step estimation of item parameters, regression coefficients β, and residual variance. This results in:

1. β estimates correctly propagate measurement uncertainty, without attenuation;
2. Each person's ability posterior is conditioned on "response + background" — information is fully utilized;
3. Subgroup comparisons (gender differences, urban-rural differences) are directly obtainable within the model, with no shrinkage bias.

The operational process for large-scale assessments such as PISA and NAEP is: IRT measurement model + latent regression (background variables are compressed via principal components and all are included) → extract **plausible values (PV)** from individual posteriors for secondary analysts to use. PVs are essentially the "multiple imputation outputs" of a latent regression model — this also explains why PV usage rules are identical to those of multiple imputation (analyze each PV separately and then combine using Rubin's rules).

## Practical Implementation: How to Do It in R

The TAM package is the standard tool for large-scale assessments:

```r
library(TAM)
# Y: background variable matrix (regressed onto ability)
mod <- TAM::tam.mml(resp = dat, Y = Y_background,
                    irtmodel = "PCM", pweights = w)
mod$beta        # latent regression coefficients
pv <- TAM::tam.pv(mod, nplausible = 10)  # extract plausible values
```

Our IRTC engine also includes latent regression and case weighting (optimized for large samples), with interface details available in the GitHub repository documentation.

**Key Points for Use:**

1. **The background model should be "wide"**: All variables potentially used in secondary analysis should be included in the latent regression (or their principal components). If a variable is not in the background model, its relationship with ability based on PV will be biased toward zero — this is the "implicit contract" of PV data: **only relationships covered by the background model are fully preserved**. Large-scale assessments compress hundreds of background variables into principal components and include them all to make this contract as wide as possible;
2. **Weights should be included simultaneously**: Sampling weights should be used in the latent regression estimation, not added afterward;
3. **Intuition for the paradox of conditioning**: Given two students with the same total score but different background variables, their PV distributions differ — beginners often find this "unfair." The key point is that PVs are designed for **group statistical inference** (reconstructing the joint distribution of the population), not for individual reporting. **PVs must never be used to score individuals**, this is its usage boundary.

## When to Use This Powerful Tool

| Scenario | Recommendation |
| --- | --- |
| Each person completes the full test, with 40+ items, and only individual scores are of interest | The two-step approach is acceptable (small error) |
| Researching the "background × ability" relationship, with moderate item count | Latent regression or SEM one-step modeling |
| Matrix-based test administration / large-scale monitoring | Latent regression + PV, industry standard, no alternative |
| Using public data such as PISA | Analyze according to PV rules (per PV + Rubin combination + matching resampling weights) |

Latent regression is a classic example of the integration of "measurement models" and "structural analysis." Understanding it gives you an additional layer of caution regarding "scores": **any point estimate is a form of information compression, and the way of compression itself carries a statistical stance.**
