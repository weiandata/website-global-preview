# A29 · CAT Adaptive Testing: Principles, Item Bank, and Implementation Conditions

> A deeper version accompanying Q21. IRT application line, for institutional decision-makers and technical staff considering CAT.

## From "Everyone Takes the Same Test" to "Each Person Gets a Unique Test"

Fixed test formats have an inherent inefficiency: the portion of items that are mismatched in difficulty and examinee ability generates almost no information. For example, top students answer easy questions, while struggling students face difficult ones, and the answers are all predictable. The idea behind computerized adaptive testing (CAT) is to turn test assembly into real-time decision-making: **after each item, update the ability estimate, and always select the next item with the most information.** The result: 40%–60% fewer items for the same level of precision, with uniform accuracy across all ability levels.

The principle was discussed in the Q&A version (the loop of starting point → estimation → item selection → termination), and this article expands on the engineering and governance aspects — **the success or failure of a CAT project lies mostly in the item bank and implementation design, not the algorithm itself.**

## Algorithm Layer: Mainstream Approaches for Four Decision Points

1. **Ability Estimation**: Use EAP during the test (stable with fewer items), and switch to WLE for final scoring (unbiased, see the ability estimation section);
2. **Item Selection Criteria**: Maximum Fisher information is the baseline; in practice, add constraints — **content balance** (items appear in proportion to the blueprint across content domains, achieved through weighted penalties or stratified selection) and **exposure control** (Sympson-Hetter or randomesque: randomly select from the top 5 items in information, preventing "star items" from being overused);
3. **Starting Point**: Start with moderate difficulty, or use background information for a warm start (note fairness review);
4. **Termination Rule**: SE threshold (recommended for equal precision termination), fixed length, or hybrid (SE reached or item limit reached). Equal precision termination is the essence of CAT — everyone is tested until the precision is sufficient.

All of these are implemented in R's mirtCAT, and there is little need to invent new algorithms at the algorithmic level.

## Item Bank Layer: The Real Heavy Investment

CAT requires much higher quality item banks than fixed tests:

- **Quantity**: The number of usable items should be at least hundreds (to support rotation under exposure control);
- **Difficulty Coverage**: Information should be sufficient across the **entire ability range** — fixed tests only need to concentrate information around key score lines, while CAT item banks should cover the ability range like a carpet. Most institutions currently lack items at the extremes;
- **Calibration Quality**: Each item's parameters must be calibrated using large sample pretesting (typically thousands of participants for 2PL/3PL models), and all items must be on the same scale (complete equating chain);
- **Continuous Supply**: Items age and leak, so item banks need an annual pipeline of new item pretesting, calibration, and entry (see the item bank construction section).

**Essential preparation before implementing CAT: item bank simulation.** Run a virtual CAT using the existing item bank on a simulated examinee group to see: the actual precision achieved at each ability level, average item length, and item exposure distribution. Simulation will brutally reveal where the item bank is lacking — **this step takes one week and can prevent the system from running out of items in a specific ability range after launch.** mirtCAT's simulation mode can complete this.

## Implementation Layer: Governance Issues Often Overlooked

1. **No Backtracking**: Most CAT systems do not allow revisiting and changing answers (changing answers can disrupt the optimality of item selection), so the examinee experience and interface design must address this upfront;
2. **Perceived Fairness**: "Why are my questions different from his, and how can the scores be comparable?" — technically, this is resolved (same θ scale), but **communication with examinees and the public** requires special preparation, especially for high-stakes exams;
3. **Interrupt Recovery and Security**: Resume testing after a network outage, encrypt item transmission, and ensure that items are not traceable after the test — these are the basics of online implementation;
4. **MST as a Compromise**: Multi-stage testing (MST) allows for a compromise — first complete a module, then route to the next based on performance — retaining most of the efficiency gains of adaptive testing while allowing backtracking and enabling content experts to review each path's test content — **in many scenarios, MST is a more reliable choice than item-by-item CAT**, and most large-scale international assessments transition to this approach.

## Decision Checklist: Should You Implement CAT?

**Suitable for**: Large and ongoing assessments (item bank investment is justified), wide ability range (adaptive benefits are significant), existing online testing infrastructure, and the ability to operate an item bank.

**Postpone for**: Fewer than a few thousand examinees per year (item bank costs are not justified, fixed tests with good item selection are sufficient); item bank not yet calibrated using IRT (first complete this step); primarily paper-based settings.

CAT is the most thorough application of IRT theory and also the most comprehensive application of an institution's measurement engineering capabilities — only when the algorithm, item bank, and implementation layers are all up to standard can it fulfill the promise of "shorter, more accurate, and fairer" testing.
