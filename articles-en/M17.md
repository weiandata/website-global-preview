# 17 · "Unidimensionality" and "Multidimensionality": Is a Questionnaire Measuring Several Things at Once?

> This is the most important assumption in IRT. If you don't know "what you're measuring," all the results that follow are just castles in the air. This article explains it in plain language.

## Set the Stage with One Sentence

**Unidimensionality = the set of items measures "the same thing"; Multidimensionality = the set of items measures "several things" at the same time.**

Most classical IRT models (1PL, 2PL, 3PL) assume you are measuring **the same thing**, and this assumption is called **unidimensionality**. It is one of the key assumptions that makes these models valid.

## A Kitchen Analogy

Imagine you are rating a pot of soup.

- If all your rating criteria are focused on "saltiness" — how much salt was added, whether it's flavorful — you are measuring **a single dimension**: saltiness. Everyone takes a taste and gives a score, and the score reflects the level of saltiness, straightforward and clear.
- But if your items include half that are about "saltiness," half that are about "spiciness," and a few about "temperature" — you are actually measuring **three different things** at the same time. At this point, adding all the scores into a "total score" is absurd: a soup that is salty but not spicy and a soup that is not salty but very spicy might end up with the same total score, yet they are completely different. **The total score mixes three different things into a single mess.**

This is the core difference between unidimensionality and multidimensionality. **When unidimensional, one θ can summarize a person; when multidimensional, you need several θs (one for each thing) to explain it properly.**

## Why Unidimensionality Is So Critical

Because IRT compresses a person into "one ability value θ." This compression is only reasonable when **all items point to the same latent variable.**

If your items are actually measuring several things, but you force a unidimensional model onto them with a single θ, what happens?

- This θ becomes a **miscellaneous mess**, and no one can say what it really represents.
- Item parameters will be estimated in a chaotic way, as the model tries to explain data that should be explained by several axes with a single axis.
- All subsequent judgments based on it (ranking, passing standards, comparisons) become unreliable.

**Using the wrong dimension is like using a ruler to measure volume — the tool and the object don't match, and no matter how precise it is, it's still wrong.**

## How to Know Whether You're Measuring One or Several Things

This isn't a decision made on a whim; it requires testing. Common methods (just for understanding, professional analysis will handle this):

- **Factor Analysis**: Check whether the set of items is explained by "one common factor" or requires several factors. This is the most commonly used first step.
- **Look at Item Content**: If the content clearly divides into several parts (e.g., a questionnaire mixing "math ability" and "reading ability" items), it's likely multidimensional.
- **Some specific unidimensionality indicators and residual checks**: After running the model, check the fit. If the unidimensional model can't fit well, it might be that the dimension is wrong.

Common practice in real-world applications: If a large questionnaire actually includes several subscales (e.g., "knowledge + attitude + behavior" three parts), it's usually **treated as unidimensional for each subscale**, calculated separately for θ, rather than forcing them into a single model.

## Is Multidimensionality Always Bad? — No

Multidimensionality isn't an error; it's just a reality. Many things are naturally multidimensional (e.g., "comprehensive literacy" indeed includes several aspects).

To deal with multidimensionality, there are two approaches:

1. **Split it up**: Break the multidimensional questionnaire into several unidimensional subscales, each measuring its own thing and reporting its own score. Simple, safe, and the most common.
2. **Use Multidimensional IRT (MIRT)**: Use a specialized multidimensional model to estimate several θs at the same time and even describe their relationships. More powerful, but more complex, data-intensive, and harder to interpret — it's an advanced technique (see A19 for a detailed discussion in the content library).

Choose which path based on your needs and resources. **If you can split it up, do so first** — this is the safe recommendation for ordinary users.

## A Common Pitfall: Don't Be Fooled by "Total Scores That Work Well"

Many people get a questionnaire with a wide range of content and habitually add up all the scores to get a total. This is dangerous when the items are actually multidimensional — the total score mixes different things into one pot, and you think you're measuring a clear thing, but you're actually measuring a confusing mix.

**Before you start calculating, ask yourself: Are these items really measuring the same thing?** This question is the starting point for all serious measurement work.

## One Final Sentence

> **Unidimensionality = a set of items measures the same thing (summarized by one θ), Multidimensionality = measures several things at the same time (requires several θs). Classical IRT assumes unidimensionality by default; using the wrong dimension makes all results invalid. When unsure, first do a factor analysis to check, and if it can be split into unidimensional subscales, do so first.**
