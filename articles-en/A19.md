# A19 · Multidimensional IRT: When Is It Needed, and How to Model It

> IRT Advanced Track. Forms part of the "Polytomous Scoring—Multidimensional—Large-Scale Computation" technical depth with A10 and A18.

## When the Single-Dimensional Assumption Fails

All of single-dimensional IRT is built on one assumption: that the set of items measures the same latent trait. However, real assessment frameworks are almost always multidimensional—science literacy includes knowledge, ability, and attitude; mathematics includes content domains and cognitive processes; digital literacy includes operation, information, and security. **When the test design itself is a multidimensional blueprint, forcing a single-dimensional model is a contradiction between method and design.**

Multidimensional IRT (MIRT) expands θ from a single number to a vector, with each item associated with one or more dimensions.

## Distinguishing Two Types of "Multidimensionality"

**Between-Item Multidimensionality (Simple Structure):** Each item belongs to only one dimension—items from dimension A, items from dimension B, clearly separated. Most large-scale assessment reports are of this type. Mathematically, it is equivalent to "several single-dimensional tests + inter-dimensional correlation structure."

**Within-Item Multidimensionality (Complex Structure):** An item simultaneously involves multiple dimensions (e.g., a math application problem requires both reading comprehension and mathematical computation). Modeling is more refined, and the complexity of interpretation and estimation is higher.

To decide which to use: look at the test blueprint (design intent) + fit comparison (data support). If the design is a simple structure, start with between-item multidimensionality.

## Why Not Just Run Several Single-Dimensional Models?

This is the most common question, and the answer is three words: **borrowing strength.**

Inter-dimensional correlations are real information—people with good science knowledge likely have good science ability as well (correlation often ranges from 0.6 to 0.8). When jointly modeling, your responses on dimension A provide information for your estimate on dimension B through the correlation structure. The benefits are immediate:

- **Boosted Reliability for Short Scales:** When each dimension has only 6–8 items, the marginal reliability of separate estimation may be only 0.55–0.65, while joint estimation can often increase this by 0.1 or more—this is often the boundary between "feasible/feasible" for dimension-based reporting;
- **Unbiased Estimation of Inter-Dimensional Correlations:** Calculating the correlation between two single-dimensional EAP scores is subject to double attenuation (measurement error and shrinkage on both sides), while MIRT directly provides the attenuated correlation at the latent variable level;
- **Essential for Matrixed Item Sets:** When each person only answers part of the dimensions, running separate single-dimensional models means each dimension uses an incomplete sample. Joint modeling uses all available information.

## Modeling Workflow

1. **Specify the Structure from the Design Blueprint** (which item belongs to which dimension), this is a confirmatory approach—standard for large-scale assessments. Exploratory MIRT (letting loadings be free) has other uses (testing whether the blueprint holds), but is not used for production scoring;
2. **Model Comparison to Justify the Choice:** Compare single-dimensional vs. multidimensional models, different dimensional structures using AIC/BIC and item fit, and write "why this structure" into the technical report;
3. **Check Inter-Dimensional Correlations:** When the correlation is >0.90, the incremental value of multidimensional reporting is questionable (almost the same thing), consider merging or using a bifactor model to answer the "total score + dimension score" question;
4. **Scoring:** Separate EAP/PV for each dimension. Note that multidimensional EAP shrinkage occurs at the vector level, and individual dimension profiles (profiles) should be interpreted cautiously—when the difference between two-dimensional scores is less than the joint standard error, the "specialization" conclusion is not valid;
5. **Reporting Scales:** Convert each dimension separately for reporting; if a total score is needed, use the designed weighted synthesis and explain the basis for the weights.

R Tools: `mirt` (`mirt(dat, model)` uses syntax to specify multidimensional structure), `TAM` (traditional for large-scale assessments, supports weights and latent regression).

## The Inevitable Computational Challenges

The number of numerical integration nodes in MIRT grows **exponentially** with the number of dimensions (15 nodes per dimension, 4 dimensions = 50,000 nodes), which is why multidimensional models frequently run out of memory on large samples. Mitigation strategies: adaptive integration with fewer nodes, QMC integration, and **dimension decomposition** and **streaming computation** under between-item multidimensionality—latter two are core designs of our open-source engine IRTC (see my article "IRT Estimation for Millions of Samples" for details). Between-item multidimensional GPCM can be completed on a regular workstation for a million samples.

## Decision Checklist

| Scenario | Recommendation |
| --- | --- |
| Single-dimensional framework / inter-dimensional correlation > 0.90 | Single-dimensional model |
| Dimension-based reporting + sufficient items per dimension + small sample | Can run separate single-dimensional models (simple and transparent) |
| Dimension-based reporting + short scale / matrixed item set | **MIRT, significant benefits** |
| Need "total score + dimension score" dual-layer reporting | Bifactor model or higher-order model |
| Within-item multidimensional design | Within-item MIRT (good for cost budgeting for interpretation) |

Multidimensionality is not always better—each added dimension must be justified with three accounts: "reporting needs + fit evidence + reliability gain." Models serve reports, and reports serve decisions.
