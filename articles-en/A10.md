# A10 · Multilevel Scoring IRT Models: PCM, RSM, GPCM (and GRM) How to Choose

> IRT Advanced Track. A Practical Guide to Modeling Likert Scales and Partially Scored Items.

## Beyond 0/1

Binary scoring (correct/incorrect) is only half of the assessment world. The other half is polytomous scoring: Likert attitude items (strongly disagree ~ strongly agree), partially scored subjective items (0/1/2 points), and frequency-based questions (never ~ always). When building IRT models for these types of data, you'll encounter four abbreviations in the literature: PCM, RSM, GPCM, and GRM. This article will clarify their relationships and provide a clear selection guide.

## A Intuitive Framework: Stairs

Imagine a 0–3 point item as three levels of stairs: from 0 to 1, from 1 to 2, and from 2 to 3. The core of polytomous scoring models is assigning a "difficulty" (threshold parameter) to **each level of the stair**:

- Stair difficulty increases (0→1 is easy, 2→3 is hard) is the norm;
- Disordered thresholds (stair difficulty in a random order) is an important diagnostic signal—often indicating that an intermediate option is essentially meaningless (e.g., "somewhat agree" and "agree" are indistinguishable), suggesting the need to merge option categories.

The difference between the four models lies in **how they model the stairs**:

**PCM (Partial Credit Model)**: A member of the Rasch family. Each level of each item has its own difficulty, but there is no discrimination parameter (all items have the same slope). Suitable for partially scored cognitive items (each item has a different scoring logic).

**RSM (Rating Scale Model)**: A member of the Rasch family. Constraints are stronger—**all items share the same set of stair structures**, differing only by an overall position. Suitable for Likert scales with consistent wording (all items use the same "strongly disagree ~ strongly agree" scale), with fewer parameters and more stability in small samples. If the wording of the options differs across items, RSM is not suitable by design.

**GPCM (Generalized PCM)**: PCM + a discrimination parameter for each item. The counterpart of 2PL to Rasch in the multilevel world. The mainstream choice for large-scale assessments (after PISA 2015, various domestic monitoring programs).

**GRM (Graded Response Model)**: A different mathematical approach (cumulative probability modeling: P(score ≥ k)), also with discrimination. The traditional mainstream in attitude/personality scale literature. GRM thresholds are naturally ordered and will not appear disordered (thus losing the diagnostic signal of disordered thresholds).

## Selection Decision Tree

```text
What is the data?
├─ Partially scored cognitive items (different scoring logic per item)
│   ├─ Sample hundreds / Rasch tradition / need full statistics → PCM
│   └─ Sample thousands+ / item quality varies → GPCM
├─ Likert scale (consistent wording)
│   ├─ Small sample + consistent wording → RSM
│   ├─ Psychological scale research tradition → GRM (good literature comparability)
│   └─ Large-scale assessment tradition → GPCM
└─ Uncertain → Fit both GPCM and GRM, compare AIC/BIC and item fit
```

Additional three experiences:

1. **GPCM vs GRM results are usually highly consistent** (θ correlation of 0.98+), selection is more driven by disciplinary tradition and project conventions, so there's no need to be anxious;
2. **Each option category needs sufficient responses** (experience baseline of dozens of respondents per category), merge sparse categories before modeling, otherwise threshold parameters will be unstable;
3. **Sample size**: Multilevel models have significantly more parameters than binary models (m-1 thresholds per item + possible discrimination), requiring a larger sample for the same level of stability. Only when the sample size exceeds a thousand can GPCM/GRM comparisons be handled with ease.

## Practical Example (mirt)

```r
library(mirt)
mod_gpcm <- mirt(dat, 1, itemtype = "gpcm")
mod_grm  <- mirt(dat, 1, itemtype = "graded")
anova(mod_gpcm, mod_grm)          # Information criterion comparison
coef(mod_gpcm, IRTpars = TRUE)    # Discrimination + thresholds for each level
plot(mod_gpcm, type = "trace")    # Category response curve: check for disordered thresholds and redundant options
itemfit(mod_gpcm)
```

Focus on the category response curve (CRC): The ideal shape is that each option dominates in a certain ability range; if an option's curve is consistently below others, it means this option has never been the most likely choice for anyone, indicating redundant option design. **This is unique questionnaire improvement information in multilevel IRT**—it can tell you whether a 5-point scale is actually used as a 3-point scale (this finding often feeds back into questionnaire revision: directly change to 4 points in the next round).

## A Reminder for Large-Scale Practice

Multilevel models have significantly higher computational demands than binary models (more integration and parameters). A GPCM with hundreds of thousands of samples may encounter memory wall issues with conventional implementations—this is a typical scenario where our open-source IRTC engine (searchable on GitHub) is optimized for streaming computation, allowing multidimensional GPCM with a million samples to run on a standard workstation. Tools are tools, but the selection logic remains unchanged: first determine the model on a subsample, then apply it to the full dataset.
