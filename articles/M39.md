# 39 · 拿到一张参数表，先看哪几列

> 跑完 IRT，软件会吐出一张密密麻麻的表格。这一篇教你像老手一样，一眼锁定最该看的几列，不被数字海洋淹没。

## 参数表长什么样

跑完模型，你会得到一张"每道题一行"的表，列大概长这样（不同软件叫法略有出入）：

| 题目 | a（区分度） | b（难度） | c（猜测） | 标准误 | 拟合指标 |
| --- | --- | --- | --- | --- | --- |
| 题1 | 1.20 | -0.5 | 0.18 | … | … |
| 题2 | 0.35 | 1.8 | 0.22 | … | … |
| 题3 | 1.85 | 0.2 | 0.05 | … | … |
| … | | | | | |

数字一多容易眼晕。别慌，老手看这张表有固定的"扫描顺序"。

## 扫描顺序第一步：先看区分度 a

**为什么先看它**：区分度直接决定一道题"有没有用"（第 10 篇）。它是判断题目好坏的第一道闸门。

**怎么看**：

- a 太小（比如低于 0.3、0.4 这个量级）→ 这题几乎不分辨人，**重点怀疑对象**，考虑淘汰。
- a 是**负的** → 警报！题目八成出了错（答案标反、反向题没翻转），必须查（第 46 篇）。
- a 适中偏大（比如 1.0 上下及以上）→ 好题的特征，分辨力强。

**一句话**：**先扫 a 那一列，把小的、负的圈出来——它们是问题题的高发区。**

## 扫描顺序第二步：再看难度 b

**为什么第二看它**：难度决定这道题"适合测谁"（第 9 篇），关系到卷子的难度覆盖。

**怎么看**：

- 把所有 b 值扫一遍，看它们**铺得均不均匀**。全挤在一头（都很难或都很简单）→ 卷子难度覆盖有问题（第 9、15 篇）。
- 看有没有 b 值**大得离谱或小得离谱**（比如超过 ±4）→ 可能是极端题，或估计出了问题（第 49 篇）。
- 对照你的**测量目标**：想测准及格线附近的人，就看有没有足够多 b 落在及格线附近的题。

**一句话**：**扫 b 那一列，看难度分布铺得开不开、有没有对准你要测的人群。**

## 扫描顺序第三步：瞄一眼猜测 c（如果有）

只有 3PL 模型才有这一列。快速扫一下：

- c 明显偏高（比如 0.4 往上）→ 这题蒙对率太高，可能干扰项太差、有破绽（第 11 篇）。
- c 跑到 0 或接近 1 的荒唐值 → 多半是样本不够、估计不稳（第 30、49 篇），别当真。

## 扫描顺序第四步：别忘了标准误

这一步最容易被跳过，但极重要（第 21 篇）。

每个参数旁边都跟着一个标准误。**标准误特别大的参数，说明"这个数其实没估准"**——哪怕它看起来是个好值，你也不能太当真。

- 某道题的 a、b 标准误都很大 → 这道题标定得不充分（可能做的人太少），它的"身份证"是模糊的。
- 报告或决策时，优先信任那些**标准误小、估得稳**的参数。

**一句话**：**别只看参数本身，要连它的标准误一起看——大标准误的漂亮数字，是虚的。**

## 扫描顺序第五步：拟合指标扫雷

最后扫一眼拟合列（第 22 篇），把拟合明显差的题标记出来。它们是"不合群"的题，要重点审查（可能违反单维、可能有依赖、可能题本身有毛病）。

## 一个"三秒扫描法"总结

拿到参数表，按这个顺序快速过一遍：

1. **a 列**：圈出太小的、负的（问题题高发区）。
2. **b 列**：看分布铺得开不开、对不对准目标人群。
3. **c 列**（若有）：瞄有没有过高或荒唐值。
4. **标准误**：给上面的判断打个"可信度折扣"。
5. **拟合列**：标记不合群的题。

圈出来的这些题，就是你下一步要重点处理的对象（第 40、41 篇教怎么判断和取舍）。

## 别犯的两个新手错

1. **只盯着难度 b 看**：很多新手以为难度最重要，其实**区分度 a 才是判断题好坏的第一指标**。难度只有合不合适，没有好坏；区分度低就是实打实的差。
2. **把参数当铁板钉钉**：忘了看标准误，把一个误差巨大的估计值当成精确真理去解读、去比较。

## 一句话收尾

> **读参数表的老手顺序：先看区分度 a（圈出太小、为负的问题题），再看难度 b（分布铺得开吗、对准目标吗），瞄一眼猜测 c，然后用标准误给所有判断打折扣，最后扫拟合列标记不合群的题。记住：a 定好坏、b 定合适、标准误定可信度。**

下一篇，我们把 a 和 b 放一起，手把手教你判断"一道题到底好不好"。
