# A19 · 多维 IRT：什么时候需要，怎么建模

> IRT 进阶线。与 A10、A18 形成"多级计分—多维—大规模计算"的技术纵深。

## 单维假设撑不住的时候

单维 IRT 的全部大厦建立在一个假设上：这组题目测量同一个潜在特质。但真实的测评框架几乎都是多维的——科学素养分知识、能力、态度；数学分内容领域与认知过程；数字素养分操作、信息、安全……**当测验设计本身就是多维蓝图时，硬套单维模型是方法与设计的自相矛盾。**

多维 IRT（MIRT）就是把 θ 从一个数扩成一个向量，每道题与一个或多个维度关联。

## 先分清两种"多维"

**题间多维（between-item，简单结构）**：每道题只属于一个维度——维度 A 的题、维度 B 的题，泾渭分明。大规模测评的分维度报告几乎都是这种。数学上等价于"几个单维测验 + 维度间相关结构"。

**题内多维（within-item，复杂结构）**：一道题同时吃多个维度（一道数学应用题既要阅读理解又要数学运算）。建模更精细，解释与估计的复杂度也高一档。

判断用哪种：看测验蓝图（设计意图）+ 拟合比较（数据支持）。设计是简单结构就从题间多维起步。

## 为什么不干脆分开跑几个单维模型？

这是最常见的疑问，答案是三个字：**借力（borrowing strength）。**

维度间相关是真实信息——科学知识好的人，科学能力大概率也不差（相关常在 0.6–0.8）。联合建模时，你在维度 A 的作答通过相关结构为你的维度 B 估计提供信息。收益立竿见影：

- **短量表的信度跃升**：每维只有 6–8 题时，单独估计的边际信度可能只有 0.55–0.65，联合估计常能抬升 0.1 上下——这常常是"分维度报告可行/不可行"的分界；
- **维度间相关的无偏估计**：拿两个单维 EAP 分数算相关会被双重衰减（两边都有测量误差+收缩），MIRT 直接给出潜变量层面的去衰减相关；
- **矩阵组卷的必需品**：每人只做部分维度的题时，单维分开跑等于每个维度都在用残缺样本，联合建模把全部信息用满。

## 建模流程

1. **从设计蓝图出发指定结构**（哪题属哪维），这是验证性思路——大规模测评的常规。探索性 MIRT（让载荷自由）另有用途（检验蓝图是否成立），但不用于生产计分；
2. **模型比较立证**：单维 vs 多维、不同维度结构的 AIC/BIC 与题目拟合对照，把"为什么是这个结构"写进技术报告；
3. **检查维度间相关**：相关 >0.90 时，多维报告的增量意义存疑（几乎是同一个东西），考虑合并或改用双因子模型回答"总分+维度分"的问题；
4. **计分**：分维度 EAP/PV。注意多维 EAP 的收缩发生在向量层面，个体的维度剖面（profile）解释要谨慎——两维分数差异小于联合标准误时，"偏科"结论不成立；
5. **报告刻度**：各维度分别转换报告分；如需总分，用设计好的加权合成并说明权重依据。

R 工具：mirt（`mirt(dat, model)` 用语法指定多维结构）、TAM（大规模测评传统、支持权重与潜在回归）。

## 绕不开的计算问题

MIRT 的数值积分节点数随维度**指数增长**（每维 15 节点、4 维 = 5 万节点），这正是多维模型在大样本上频繁爆内存的原因。缓解路线：降节点数的自适应积分、QMC 积分，以及题间多维结构下的**维度分解**与**流式计算**——后两者是我们开源引擎 IRTC 的核心设计（详见我的《百万级样本的 IRT 估计》一文），题间多维 GPCM 在百万样本上可于普通工作站完成。

## 决策速查

| 情形 | 建议 |
| --- | --- |
| 框架单维/维度相关>0.9 | 单维模型 |
| 分维度报告 + 每维题量充足 + 小样本 | 可分开单维（简单透明） |
| 分维度报告 + 短量表/矩阵组卷 | **MIRT，收益显著** |
| 要"总分+维度分"双层报告 | 双因子模型或高阶模型 |
| 题内多维设计 | within-item MIRT（解释成本预算好） |

多维不是越多越好——每加一维，都要用"报告需求 + 拟合证据 + 信度增量"三笔账来辩护。模型为报告服务，报告为决策服务。
