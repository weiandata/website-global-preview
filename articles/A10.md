# A10 · 多级计分 IRT 模型：PCM、RSM、GPCM（和 GRM）怎么选

> IRT 进阶线。李克特量表与部分给分题建模的实操指南。

## 0/1 之外的世界

二级计分（对/错）只是测评世界的一半。另一半是多级计分：李克特态度题（非常不同意~非常同意）、部分给分的主观题（0/1/2 分）、行为频率题（从不~总是）。给这些数据建 IRT 模型，你会在文献里撞见四个缩写：PCM、RSM、GPCM、GRM。本文把它们的关系一次说清，并给选择规则。

## 一个直觉框架：台阶

把一道 0–3 分的题想象成三级台阶：从 0 到 1、从 1 到 2、从 2 到 3。多级计分模型的核心就是给**每级台阶**一个"难度"（阈值参数）：

- 台阶难度递增（0→1 容易，2→3 难）是常态；
- 台阶难度**乱序**（disordered thresholds）是重要诊断信号——常意味着某个中间选项形同虚设（比如"比较同意"和"同意"没人能分清），提示合并选项类别。

四个模型的区别，就在"怎么给台阶建模"：

**PCM（Partial Credit Model）**：Rasch 家族。每道题的每级台阶都有自己的难度，无区分度参数（各题斜率同）。适合部分给分的认知题（每题的给分逻辑不同）。

**RSM（Rating Scale Model）**：Rasch 家族。约束更强——**所有题共享同一组台阶结构**，各题只差一个整体位置。适合选项措辞完全统一的李克特量表（所有题都是同一套"非常不同意~非常同意"），参数省得多，小样本更稳。若各题选项措辞不同，RSM 从设定上就不合适。

**GPCM（Generalized PCM）**：PCM + 每题自己的区分度参数。2PL 之于 Rasch 的关系，在多级世界的对应物。大规模测评（PISA 2015 后、各类国内监测）的主流选择。

**GRM（Graded Response Model）**：不同的数学路线（累积概率建模：P(得分≥k)），同样带区分度。态度/人格量表文献的传统主流。GRM 的阈值天然有序，不会出现乱序（因此也失去了"乱序作为诊断信号"的功能）。

## 选择决策树

```text
数据是什么？
├─ 部分给分认知题（各题给分逻辑不同）
│   ├─ 样本数百 / Rasch 传统 / 要充分统计量 → PCM
│   └─ 样本数千+ / 题目质量参差 → GPCM
├─ 李克特量表（选项措辞统一）
│   ├─ 小样本 + 措辞完全一致 → RSM
│   ├─ 心理量表研究传统 → GRM（文献可比性好）
│   └─ 大规模测评传统 → GPCM
└─ 不确定 → GPCM 和 GRM 都拟合，比较 AIC/BIC 与题目拟合
```

补充三条经验：

1. **GPCM vs GRM 的结果通常高度一致**（θ 相关 0.98+），选择更多受学科传统与项目惯例驱动，不必焦虑；
2. **每个选项类别都要有足够作答**（经验底线每类几十人以上），稀疏类别先合并再建模，否则阈值参数飘；
3. **样本量**：多级模型参数比二级多得多（每题 m-1 个阈值 + 可能的区分度），同等稳定性需要更大样本，千人以上再谈 GPCM/GRM 比较从容。

## 实操示例（mirt）

```r
library(mirt)
mod_gpcm <- mirt(dat, 1, itemtype = "gpcm")
mod_grm  <- mirt(dat, 1, itemtype = "graded")
anova(mod_gpcm, mod_grm)          # 信息准则比较
coef(mod_gpcm, IRTpars = TRUE)    # 区分度 + 各级阈值
plot(mod_gpcm, type = "trace")    # 类别反应曲线：看乱序与虚设选项
itemfit(mod_gpcm)
```

重点看类别反应曲线（CRC）：理想形态是每个选项在某个能力段"各领风骚"；某选项的曲线全程被压在别人下面 = 这个选项从未成为任何人最可能的选择 = 选项设计冗余。**这是多级 IRT 独有的问卷改进信息**——它能告诉你 5 点量表是不是实际上只在当 3 点用（这个发现常反哺问卷修订：下轮直接改 4 点）。

## 大规模实践的一句提醒

多级模型的计算量显著高于二级（积分与参数都多）。数十万级样本的 GPCM，用常规实现可能遭遇内存墙——这正是我们开源 IRTC 引擎（GitHub 可搜）针对流式计算做优化的典型场景，百万样本多维 GPCM 在普通工作站可跑。工具是工具，选型逻辑不变：先在子样本上定模型，再上全量。
