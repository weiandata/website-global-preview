# A29 · CAT 自适应测验：原理、题库与实施条件的全面盘点

> 配套 Q21 的深度版。IRT 应用线，面向考虑上 CAT 的机构决策者与技术人员。

## 从"人人同卷"到"千人千卷"

固定卷子的测量效率有个天然浪费：题目难度与考生能力错配的部分几乎不产生信息——学霸做简单题、后进生做难题，答案都在意料之中。计算机自适应测验（CAT）的思路是把组卷变成实时决策：**每答一题就更新能力估计，下一题永远挑当前最有信息量的。**效果：同等精度下题量省 40%–60%，且全能力段精度均匀。

原理在问答版里讲过（起点→估计→选题→终止的循环），本文把工程与治理层面展开——**CAT 项目的成败，九成不在算法，在题库和实施设计。**

## 算法层：四个决策点的主流方案

1. **能力估计**：作答过程中用 EAP（少题时稳定），终测分可换 WLE 复核（无偏，见能力估计篇）；
2. **选题准则**：最大 Fisher 信息是基线；实务必加约束——**内容均衡**（各内容域按蓝图比例出现，用加权惩罚或分层选题实现）与**曝光控制**（Sympson-Hetter 或 randomesque：从信息量前 5 的题里随机挑，防止"明星题"被用穿）；
3. **起点**：中等难度起步，或用背景信息热启动（注意公平性审查）；
4. **终止规则**：SE 阈值（等精度终止，推荐）、定长、或混合（SE 达标或题量到上限）。等精度终止是 CAT 的精髓——每个人都被测到"够准为止"。

这些在 R 的 mirtCAT 里都有现成实现，算法层面几乎没有需要发明的东西。

## 题库层：真正的重资产

CAT 对题库的要求远高于固定卷：

- **数量**：可用题量至少数百道起步（要支撑曝光控制下的轮换）；
- **难度覆盖**：信息量要在**整个能力范围**上充足——固定卷只需在关键分数线附近堆信息，CAT 的题库难度分布要像地毯一样铺满。多数机构现有题库在两端严重缺题；
- **标定质量**：每题参数都要经过大样本预试标定（2PL/3PL 常用样本千人级），且全部在同一刻度上（等值链完整）；
- **持续补给**：题目会老化、会泄露，题库需要年度级的新题预试-标定-入库流水线（见题库建设篇）。

**上 CAT 之前的必做功课：题库仿真。**用现有题库对模拟考生群体跑虚拟 CAT，看：各能力段的实际达到精度、平均题长、题目曝光分布。仿真会残忍地告诉你题库缺口在哪——**这一步花一周，能避免上线后整个系统在某个能力段"无题可选"的事故。**mirtCAT 的仿真模式即可完成。

## 实施层：容易被忽略的治理问题

1. **不可回改**：多数 CAT 不允许回头改答案（改答案会破坏选题逻辑的最优性），考生须知与界面设计要提前处理这个体验落差；
2. **公平性感知**："我做的题和他不一样，凭什么分数可比？"——技术上早已解决（同刻度的 θ），但**面向考生与社会的沟通**需要专门准备，高利害考试尤其如此；
3. **中断恢复与安全**：断网续考、题目传输加密、考后题目不可回溯——线上实施的基本功；
4. **变式测验（MST）作为折中**：多阶段测验（先做一个模块，按表现路由到下一模块）保留了自适应的大部分效率收益，同时允许回改、便于内容专家审查每条路径的卷面——**很多场景下 MST 是比逐题 CAT 更稳妥的选择**，国际大型测评的机考转型多走这条路。

## 决策清单：你该不该上 CAT

适合：考量大且持续（题库投资摊得开）、能力范围宽（自适应收益大）、已有机考基础设施、有题库运营能力。

暂缓：年考量几千人以下（题库成本摊不平，固定卷+好的组卷设计够用）；题库尚未 IRT 标定（先补这一课）；纸笔为主的场景。

CAT 是 IRT 理论优势兑现得最彻底的应用，也是对机构测量工程能力要求最全面的应用——算法、题库、实施三层都及格，它才能兑现"更短、更准、更公平"的承诺。评估自身条件或规划技术路线时，欢迎交流。
