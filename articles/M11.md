# 11 · 那条"抬不起头"的曲线——猜测参数 c 是怎么回事

> 你可能注意到，有些 ICC 的左端不贴着地面，而是悬空抬起一小截。这一篇讲清楚这个"小台阶"是什么，以及它在提醒你什么。

## 从一个常识开始：选择题能蒙

四选一的选择题，哪怕你一个字都不认识，闭着眼睛蒙，也有 25% 的概率撞对。

这个"再不会也能蒙对"的保底概率，在 IRT 里有个专门的参数来描述，叫**猜测参数**，符号是 **c**（也叫下渐近线）。

前面讲的 1PL、2PL 模型都假装蒙对不存在（曲线左端一路贴到 0）。但对有蒙对空间的选择题，这个假设不成立——于是 3PL 模型登场，专门加进这个 c。

## 它在图上长什么样

回忆标准的 ICC：左端（低水平的人）答对概率贴着 0，因为"不会就是不会"。

现在给它加上猜测。低水平的人虽然不会做，但可以蒙，所以他们的答对概率不会真的掉到 0，而是**停在一个小台阶上**——这个台阶的高度，就是 c。

于是整条曲线的左端"抬起了头"：不再从地板出发，而是从半空中的一个小平台出发，再往右上方爬升。一道四选一的题，这个台阶大约在 0.25 左右（当然实际会有出入，好的干扰项能压低它，太诱人的干扰项反而可能让它更低）。

**画面感**：普通 ICC 是从地面起跑的运动员；带猜测的 ICC 是从一个小台子上起跑的运动员——起点被垫高了。

## 为什么必须正视"蒙对"

因为忽略它会**高估低水平者的能力**。

设想一道很难的选择题。有个水平很低的人蒙对了。如果模型不考虑蒙对，它会一脸认真地想："他连这么难的题都做对了，水平一定不低啊！"于是把他的能力估高了。这就冤枉了——人家纯粹是运气好。

3PL 模型加进 c 之后，就学会了"打折"：碰到低水平者做对难题，它会想"这有可能是蒙的"，从而不轻易抬高能力估计。**c 的作用，就是给模型装上一副‘防蒙滤镜’。**

## 但 c 是个"难伺候"的参数

必须诚实地说：c 是三个参数里最不稳定、最难估准的一个。

原因很直白：**估计 c 靠的是"低水平的人在这道题上的表现"**。可要是你的被试里低水平的人很少，或者这道题对所有人都不算太难，那"低水平者怎么蒙这道题"的数据就严重不足，机器只能连蒙带猜地给个 c，结果经常飘忽不定，甚至跑出离谱的值。

所以实务中常见几种处理：样本不够时干脆不用 3PL（退回 2PL）；或者给 c 设一个合理的先验（相当于告诉机器"四选一的话，c 大概率在 0.2 附近，别乱猜"）。这些技巧第 30、49 篇会再提。

## 一个常被问的问题：李克特量表有 c 吗

一般没有。

猜测参数是**对错题（尤其选择题）**的专利，因为那里存在"蒙一个答案"的动作。而"非常同意/同意/一般/不同意/非常不同意"这种态度量表，你不是在"蒙对一个正确答案"，而是在表达一个程度，没有"标准答案可蒙"这回事。所以处理李克特量表的 GRM 那类模型，不带猜测参数。

**一句话**：有"标准答案可以蒙"的题，才需要考虑 c。

## 读图时怎么用 c

看到一条 ICC 左端抬得老高（比如台阶都到 0.4 了），你该起点疑心：

- 是不是这道题的**干扰项太差**，一眼就能排除掉两个，实际成了"二选一"，蒙对率自然高？
- 是不是这道题有**明显破绽**，能靠常识或排除法猜到？
- 还是纯粹**样本不够、c 估歪了**？

反过来，c 接近 0，说明这道题几乎蒙不对——要么是填空/主观题，要么是干扰项设计得极好。

## 一句话收尾

> **那条"抬不起头"的曲线，左端的小台阶就是猜测参数 c——描述"再不会也能蒙对"的保底概率。它给模型装上防蒙滤镜，避免高估低水平者；但它最难估准，样本不够时宁可不用。**

下一篇，我们把 ICC 变个形，认识 IRT 里另一条超实用的曲线——信息函数，它告诉你"这道题在哪段人群最好用"。
