# A13 · 测验等值与链接：让不同卷子的分数说同一种语言

> 配套 Q16 的深度版。连续性监测项目的核心技术。

## 可比性是设计出来的

任何跨卷、跨年比较的合法性，都建立在一个技术前提上：两套分数已被放到同一把尺上。这个过程叫等值（equating）/链接（linking）。本文讲清它的设计、方法与工程要点——以及为什么说**等值是设计阶段的事，数据收完再想已经晚了**。

## 前提：必须有桥

两份卷子测两批人，没有任何交集，等值在数学上不可能。三种搭桥设计：

**1. 锚题设计（NEAT，最常用）**：两卷共享一组锚题。工程要求：

- 锚题量：全卷的 20%+（少于此，链接误差主导一切）；
- 代表性：锚题的内容分布、难度分布应是全卷的缩影（全用中等难度题当锚，两端等不准）；
- 位置稳定：同一道锚题在两卷中的位置尽量一致（位置效应会伪装成难度漂移）；
- **保密**：锚题泄露 = 等值报废 = 趋势结论报废。

**2. 共同被试设计**：同批人做两卷（或随机半卷交叉）。适合小规模标定，疲劳与顺序效应要用平衡设计控制。

**3. 随机组设计**：同总体随机分派做不同卷。逻辑最干净，对"随机分派"的执行质量要求极高。

## 方法：两大家族

**经典等值**（观察分数层面）：

- 均值等值/线性等值：把分布的均值（和标准差）对齐，简单粗暴；
- 等百分位等值：把两卷的百分位一一对应（78 分若在 A 卷是第 63 百分位，就映射到 B 卷的第 63 百分位分数），灵活但需要大样本平滑。
经典方法直观、无模型假设，但对样本代表性敏感，且给不出"题目层面"的诊断。

**IRT 等值**（参数层面）——大规模测评主流：

核心是把两卷题目参数放上同一刻度，路线有三：

1. **固定锚参数**：新卷标定时锚题参数固定为题库既有值，新题参数自动落到既有刻度上。题库运营的日常做法；
2. **分别标定 + 转换**：两卷各自标定，再用锚题参数估计线性转换系数（均值-均值、均值-标准差，或更稳健的 Stocking-Lord / Haebara 特征曲线法——后者对个别异常锚题更不敏感，是推荐默认）；
3. **同时标定（concurrent calibration）**：两卷数据合并、一次估计，缺答部分按设计缺失处理。统计上最有效，但一颗老鼠屎坏一锅粥（任何一卷的数据问题污染全体），大项目常用"同时标定为主 + 分别标定交叉验证"。

R 工具：equate（经典方法）、plink/SNSequate（IRT 转换）、mirt/TAM（多组同时标定）。

## 工程要点：等值最容易死在哪

**1. 锚题漂移。**锚题参数在两次施测间显著变化——原因可能是课程改革（内容变熟）、泄题、位置变动。**等值前必须做漂移检测**（参数差异检验/位移图），漂移锚题剔除出等值集。剔除后锚题不够怎么办？没有好办法——所以设计时要留冗余。

**2. 链条误差累积。**A→B→C→D 逐年链接，每步误差累积，五年后趋势里混着多少链接噪声说不清。对策：设计"回锚"机制（隔几年与基准年直接链接）、报告链接误差（把等值不确定性纳入趋势的标准误——大规模测评报告里的 linking error 项就是它）。

**3. 构念漂移。**十年间"数字素养"的内涵变了，题目老化了——这时候即使统计等值完美，"分数可比"在实质意义上也已经存疑。长期项目要定期做框架修订与桥接研究（新旧框架并行施测一轮），这超出统计技术，是项目治理问题。

## 给项目负责人的检查单

- [ ] 施测前：锚题布局设计（数量/代表性/位置）+ 保密预案
- [ ] 标定时：锚题漂移检测 → 剔除 → 转换方法（默认 Stocking-Lord）+ 敏感性比较
- [ ] 报告时：链接误差纳入趋势推断；等值方法与锚题处理完整写入技术文档
- [ ] 长期：回锚设计 + 框架修订的桥接预案
